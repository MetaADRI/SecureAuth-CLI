What is a TOTP code?

A time-based one-time password that changes every 30 seconds. Only your phone and SecureAuth know the secret key.

Why authenticator apps?

They generate codes offline on your device. No SIM swap, no SMS interception, no phishing can steal your 2FA code in real time.

What if I lose my phone?

SecureAuth provides recovery codes during setup. Store them safely — they are your backstage pass to regain access.

Login Flow

Logging In Securely

Every login follows a security chain. Each step exists to stop a specific attack.

1

Credentials Sent

Your email and password are sent over an encrypted HTTPS connection. No one on the network can read them.

2

Server Verification

SecureAuth checks your credentials against the bcrypt hash. Even the server never sees your raw password.

3

Temporary JWT Issued

A temporary token is generated with a 5-minute expiry. It is a limited key — it can only be used for the next step.

4

Why a temporary token?

If an attacker intercepts the temp token, it expires in 5 minutes and cannot access any protected data. It is a one-purpose key: complete 2FA.

Security Layers

How SecureAuth Protects You

Each card explains a security feature — click to expand and learn more.

Two-Factor Authentication

Requires a second verification factor (TOTP code) in addition to your password.

Threat prevented: Credential theft, phishing, replay attacks

Real world: Google, GitHub, and banks all use TOTP 2FA

JWT Sessions

Stateless tokens with embedded user identity and expiry. No server-side session storage needed.

Threat prevented: Session hijacking, CSRF, token forgery

Enterprise usage: Used by Auth0, Okta, and AWS IAM

Account Lockout

After 5 failed login attempts, the account is locked for 30 minutes.

Threat prevented: Brute force attacks, credential stuffing

Real world: Standard in Windows Active Directory, AWS IAM

Rate Limiting

Limits requests to 5 per 15 minutes per IP on login endpoints.

Threat prevented: Automated attacks, DDoS, API abuse

Enterprise usage: Stripe API, GitHub API, Cloudflare

Application Flood Protection

6-layer protection: IP banning, connection limits, request size checks, malformed request detection, and progressive slowdown.

Threat prevented: Layer 7 DDoS, slow loris, buffer overflow

Enterprise usage: Comparable to AWS Shield and Cloudflare
